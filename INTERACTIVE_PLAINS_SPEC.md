# 인터렉티브 평원 — 통합 설계서

> 작성일: 2026-02-23
> 기반: Project Nemo (2025-09) + 인터렉티브 평원 구상 + 나만의 집 꾸미기 + 현재 대화
> 배포: fetreegame (Next.js + Vercel)

---

## 1. 비전

유저가 그리고, 세계가 살아나고, 채널이 성장하는 **참여형 인터렉티브 플랫폼**.

시청자가 곧 참여자, 참여자가 곧 콘텐츠 제작자.
개별 작품이 모여 생태계가 되고, 생태계가 세계관이 되며,
세계관이 영상과 게임의 원천이 되는 **자급자족 순환 구조**.

### 핵심 원칙
- **접근성 최우선** — 설치 없이 웹, 익명, 규칙 최소화
- **단순하게 시작, 점진적 확장** — Nemo의 교훈: 과설계 금지
- **클라이언트/서버 분리** — 비동기 서버, 언제든 교체 가능
- **우열 없는 구조** — 익명성 + 랜덤 부여로 실력 비교 제거
- **유저 기여가 곧 콘텐츠** — 작품, 이펙트, 사운드 모두 유저 생산

## 2. 단계별 로드맵

### Phase 1 — 허허벌판 (v1, MVP)
> 들어가서 그리고 놓으면 끝.

- 평원: 수평 스크롤 평지, pixi-viewport 카메라, 컬링
- 캐릭터: 기본 도형 1개, 좌우 이동만
- 그림판: PixiJS Graphics, 흑백, 펜/지우개/굵기
- 저장: 벡터 행동로그(JSON) → 서버 자동 배치
- 배포: 만들자마자 공개

### Phase 2 — 색이 찾아오다 (v2)
> "평원에 색이 찾아왔습니다" — 쇼츠 한 편.

- 그림판 색상 도입 (제한 팔레트 → 점진 확장)
- 캐릭터 커스터마이징 (아틀라스 기반 외형 그리기)
- 구름 댓글 (메시지가 하늘에 흘러감, Nemo 계승)
- 신고 기능 + 이용약관

### Phase 3 — 생태계 (v3)
> 랜덤 주제 부여, 개별 작품이 생태계의 일부가 됨.

- 테마 도입: 동물 / 식물 / 곤충 / 구조물 / 자연요소
- 랜덤 주제 부여: 유저가 제안 → 풀에서 랜덤 할당
- 카테고리별 행동 규칙 (동물=움직임, 식물=제자리, 곤충=비행)
- AABB 겹침 방지, 유저 배치 자유도 도입

### Phase 4 — 나만의 공간 (v4)
> 나만의 집 꾸미기 통합. 평원 위에 개인 공간이 생김.

- 개인 필드: 벽, 바닥, 가구, 펫 배치
- 방문 기능: 다른 유저 공간 탐방, 방문 기록
- 방명록: 방문자 메시지
- 선물/우편: 하루 1회 도트 선물 (미리보기 → 수락/거절)

### Phase 5 — 살아있는 세계 (v5)
> 평원이 세계관으로 확장. 지역이 열리고, 공동 목표가 생김.

- 새 지역 해금 (평원 → 숲 → 호수 → 마을)
- 지역 해금 조건 = 유저 작품 누적 (공동 목표)
- 시즌 이벤트 / 테마파크 (Nemo의 모나의 나무 계승)
- 모션 스캔 + 관절 추론 → 캐릭터 자동 리깅
- 이펙트/사운드 유저 기여 (메일 제출 → 큐레이션)

### Phase 6 — 수익화 (v6)
> 유저가 만든 세계에서 플레이하는 게임.

- 유저 작품을 게임 에셋으로 활용
- 게임 판매 / 후원
- "내가 기여한 세계에서 플레이하는 게임" = 구매 동기

## 3. 기능 상세 (v1 MVP)

### 3.1 평원
- PixiJS v8 캔버스, 수평 스크롤 평지
- pixi-viewport로 카메라 제어
- 뷰포트 밖 오브젝트 렌더링 제외 (컬링)
- 접속 시 서버에서 기존 작품 fetch → 평원에 렌더링

### 3.2 캐릭터
- 기본 캐릭터 1개 (단순 도형)
- 좌우 이동만, 중력 없음
- PC: 키보드 방향키
- 모바일: 가상 조이스틱 (nipplejs)

### 3.3 그림판
- PixiJS Graphics 기반
- 흑백 (색상 선택 없음)
- 도구: 펜, 지우개, 굵기 조절
- 벡터 방식 — 행동로그(좌표 배열) 저장
- 그린 그대로 평원에 바로 배치 (변환 없음)

### 3.4 저장 + 배치
- 그림 완성 → JSON으로 서버에 저장
- 서버가 자동으로 빈 자리에 순서 배치
- 겹침 방지: AABB 기반 (서버 측 검증)
- 유저 배치 선택 없음 (v3에서 도입)

### 3.5 모바일 컨트롤러

```
┌─────────────────────┐
│                     │
│       평원 뷰       │
│                     │
├──────────┬──────────┤
│ 조이스틱/ │ [좌클릭] │
│ 터치패드/ │          │
│ 펜 (토글) │ [우클릭] │
└──────────┴──────────┘
```

| 모드 | 좌측 영역 | 우측 영역 | 용도 |
|------|----------|----------|------|
| 조이스틱 | 방향 입력 | 좌클릭/우클릭 | 캐릭터 이동 |
| 터치패드 | 상대좌표 delta | 좌클릭/우클릭 | 커서 탐색 |
| 펜 | 절대좌표 + 오프셋 (길이 조절) | 좌클릭/우클릭 | 그리기 |

## 5. API 설계

fetreegame Next.js App Router에 API 라우트 추가.

### v1

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/plains` | 전체 작품 목록 반환 |
| POST | `/api/plains` | 새 작품 저장 + 자동 배치 |

### v3+ 확장

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/plains/themes` | 현재 테마 + 주제 풀 |
| POST | `/api/plains/assign` | 랜덤 주제 부여 |
| POST | `/api/plains/report` | 작품 신고 |

### v4+ 확장

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/space/:id` | 개인 공간 조회 |
| POST | `/api/space` | 공간 생성/수정 |
| POST | `/api/space/:id/gift` | 선물 보내기 |
| POST | `/api/space/:id/guestbook` | 방명록 작성 |

## 8. 콘텐츠 순환 구조

```
쇼츠 (평원 걷기 녹화, 제작 비용 ~0)
  → 유입 (링크 클릭, 설치 없이 웹 접속)
    → 참여 (익명, 그리고 놓으면 끝)
      → 작품 축적 (평원 성장)
        → 쇼츠 소재 증가 (자급자족 루프)
          → 유저 증가
            → 테마/랜덤 도입 (생태계 균형)
              → 세계관 형성 (지역 확장)
                → 게임 에셋 활용
                  → 게임 판매/후원 (수익화)
```

### 유저 기여 채널 (단계적 확장)

| 단계 | 기여 방식 | 경로 |
|------|----------|------|
| v1 | 그림 | 웹 그림판 → 즉시 등록 |
| v3 | 테마 작품 | 랜덤 주제 부여 → 웹 그림판 |
| v4 | 집 꾸미기 | 개인 공간 에디터 |
| v5 | 이펙트/사운드 | 메일 제출 → 큐레이션 반영 |

## 9. 안전 장치

### 공통
- 이용약관: "타인의 IP 침해 금지" 명시
- 알게 된 후 조치 원칙 (DMCA 대응)
- 초기: 수동 관리 (작품 수 적음)
- 성장 후: 신고 기능 + 자동 숨김

### 개인 공간 (v4+, Nemo 아틀라스 시스템 계승)
- 메일 + PIN 간단 인증
- PIN은 SHA256 해시 저장
- 신고 시 비공개 처리 (데이터 삭제 안 함)
- 작성자는 본인 인증 후 열람/백업/재공개/삭제 가능
- UX 톤: 금지보다 안내 중심 — 창작 의욕을 꺾지 않음

## 10. 차별점 — 왜 따라할 수 없는가

1. **생태계 축적** — 유저 작품이 쌓인 역사는 복제 불가
2. **파이프라인** — Animate + JSFL AI + PixiJS + 모션 리깅, 기술 스택 자체가 해자
3. **채널 연결** — 평원이 단독 서비스가 아니라 유튜브 채널의 세계관 일부
4. **순환 구조** — 쇼츠 → 유입 → 참여 → 에셋 → 게임 → 수익의 전체 루프
5. **도메인 경험** — Nemo 과설계 교훈, Pygame 웹 호환 삽질, ECS_FW 아키텍처 설계

## 11. Nemo에서 계승하는 것

| Nemo 설계 | 평원에서의 형태 | 도입 시점 |
|-----------|---------------|----------|
| 구름 댓글 | 메시지가 하늘에 흘러감 | v2 |
| 개인 공간 꾸미기 | 나만의 집 | v4 |
| 방문/방명록/선물 | 소셜 기능 | v4 |
| 모나의 나무 (글로벌 협력) | 지역 해금 공동 목표 | v5 |
| 테마파크 (공동 건설) | 시즌 이벤트 | v5 |
| 해상도/색상 해금 | 별도 미니게임으로 분리 가능 | 미정 |

| 펫 시스템 | 캐릭터 커스터마이징 확장 | v5+ |

### Nemo에서 버리는 것
- Pygame + Pygbag (→ PixiJS + Next.js)
- Google Sheets 서버 (→ Next.js API + DB)
- 픽셀 강제 (→ 벡터 자유 그리기)
- 초기부터 복잡한 시스템 (→ 최소 기능부터)

## 12. 설계 철학

> "기술이 아니라 생태계가 해자."

- 기능은 단순해도 하나하나가 기분 좋게 동작하는 것
- 참여와 소비의 경계를 의식 못하게 만드는 것
- 실력이 아닌 참여 자체에 가치가 생기는 구조
- 빈 평원부터 채워지는 과정 자체가 콘텐츠
